# Type Data Bundles

`stdlib-*-protos.json` files are the prototype bundles imported into
`.glaurung` projects by `glaurung.llm.kb.xref_db.import_stdlib_prototypes`.

## Windows API Generation

`stdlib-winapi-protos.json` is generated by:

```bash
glaurung types sync
```

The command is the only path that downloads metadata. Normal binary analysis
uses the checked-in bundle and stays offline by default.

Pinned sources live in `windows-api-sources.lock.json`:

- `Microsoft.Windows.SDK.Win32Metadata` for Win32/user-mode prototypes.
- `Microsoft.Windows.WDK.Win32Metadata` for WDK/kernel-adjacent prototypes.

Curated semantics live in `overlays/windows-api-semantics.json`. Keep only
facts that WinMD does not carry well there: buffer/source/length roles,
allocator/free relationships, risk tags, and CRT/kernel prototypes missing
from Microsoft metadata.

Bundle import preserves the metadata needed by bug-hunting tools:
`module`, `calling_convention`, `source`, `source_kind`,
`source_package`, `source_version`, `confidence`, provenance JSON,
semantic provenance JSON, semantic risk tags, and parameter roles all
round-trip through `.glaurung` project files. Callers can query this via
`get_function_prototype`, `get_function_semantics`,
`list_function_prototypes_by_risk_tag`, and
`list_function_prototypes_by_param_role`.

Project auto-load is format-aware. `PersistentKnowledgeBase.open(...,
auto_load_stdlib=True)` loads WinAPI bundles only for PE binaries; ELF and
Mach-O projects get the libc baseline. Loaded bundles are journaled in
`stdlib_bundle_loads` and surfaced through
`PersistentKnowledgeBase.list_stdlib_bundle_loads`.

PE import tools use the generated catalogue directly. `map_pe_iat` enriches
IAT or fallback symbol-import rows with module/prototype/source metadata,
confidence, risk tags, and parameter roles. `list_suspicious_imports` uses
the same catalogue to attach prototype metadata to bucketed imports and to
surface overlay risk tags such as `ioctl` and `kernel_boundary`.

Local SDK/WDK headers are opt-in augmentation sources:

```bash
glaurung types sync --offline --header /path/to/header.h --clang-arg -I/path/include
```

Those prototypes are parsed from Clang AST JSON, merged at lower confidence,
and recorded in the manifest under `header_results`.

Generated provenance is written to:

- `generated/MANIFEST.json`

The manifest records package versions, source URLs, SHA256 values, output
hashes, and optional lower-confidence source families such as local headers,
PDB symbol-server data, phnt/System Informer, ReactOS, Wine, and Ghidra/GDT
archives.
