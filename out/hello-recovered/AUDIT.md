# Audit — hello

The tree recovers a 12-function hello-world ELF but shows several tree-scale smells: std_ctype_char_do_widen is almost certainly an invented/misclassified function (conf 0.20, 0 assumptions, empty summary) placed in an application module instead of an imports list; main and stream_endl carry unusually high assumption counts (9 and 7) that will propagate signature errors through the whole callgraph; vector_string_dtor has no visible constructor counterpart and may be dead or EH-only; main_cold_eh_cleanup indicates a C++ unwind path that the rewrite appears to have partly silenced; _start is oddly low-confidence for a standard crt entry; and the module split into hello/core/ is not justified by the callgraph. No blockers, but four high-severity findings prevent a pass.

- **[medium]** module_coherence @ project layout (hello/core/core.c vs src/*): Module decomposition mixes two naming conventions: a flat src/ tree (main.c, vector_string.c, iostream_support.c) and a nested hello/core/core.c. With only 12 functions (9 of them in crt/ or src/), promoting a single helper (helloworld_print_message) plus a likely-spurious std_ctype_char_do_widen into a nested hello/core/ package is not justified by the callgraph.  → _Flatten hello/core/core.c into src/ (e.g., src/hello_core.c) unless the binary shows a distinct translation unit boundary (separate .o, distinct file symbol in debug info)._
- **[medium]** signature_mismatch @ vector_string_dtor (src/vector_string.c): A vector<string> destructor is present (conf 0.86) but no corresponding constructor, push_back, or allocator function exists in the 12-function list. If only invoked from an .eh_frame cleanup in main, confirm object layout and 'this' type matches main's stack frame; otherwise flag as dead._
- **[medium]** dead_code @ vector_string_dtor (src/vector_string.c): For a hello-world program with no visible vector construction function, the vector<string> destructor may be unreachable from any normal control flow and only referenced via an unwind table that is itself never triggered.  → _Confirm reachability via call sites in main and main_cold_eh_cleanup. - **[medium]** missing_error_path @ main / main_cold_eh_cleanup (src/main.c): main_cold_eh_cleanup exists (suggesting a C++ exception cleanup path was split out) but the recovered main is at low confidence and stream_endl/helloworld_print_message have no visible throw/cleanup linkage in the summary.
